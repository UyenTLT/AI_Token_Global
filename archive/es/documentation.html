Primeros Pasos

AI Token King proporciona una pasarela de API unificada a múltiples modelos de IA — GPT, Claude, Gemini, DeepSeek y Qwen — a través de un único endpoint y saldo de tokens. Esta documentación cubre todo lo que necesitas para integrar AI Token King en tus aplicaciones.

Autenticación

Todas las solicitudes de API requieren una clave API pasada como token Bearer en el encabezado de Autorización. Puedes generar claves API desde tu panel en Settings → API Keys.

Authorization: Bearer atk_your_api_key_here

Nunca expongas tu clave API en código del lado del cliente o repositorios públicos. Usa variables de entorno y solicitudes del lado del servidor.

Tu Primera Solicitud

Aquí hay un ejemplo mínimo para obtener una completación de GPT-4o usando la API de AI Token King:

curl https://api.aitokenking.com/v1/chat/completions \ -H "Content-Type: application/json" \ -H "Authorization: Bearer atk_your_key" \ -d '{ "model": "gpt-4o", "messages": [ { "role": "user", "content": "Explain AI tokens in one sentence." } ] }'

La respuesta sigue el formato compatible con OpenAI, lo que facilita migrar integraciones existentes:

{ "id": "chatcmpl-abc123", "object": "chat.completion", "model": "gpt-4o", "choices": [{ "message": { "role": "assistant", "content": "AI tokens are the small text units that language models use to process and generate text, with usage charged per token consumed." }, "finish_reason": "stop" }], "usage": { "prompt_tokens": 12, "completion_tokens": 28, "total_tokens": 40 } }

Referencia de la API

La API de AI Token King es compatible con OpenAI. Si ya usas el SDK de OpenAI, puedes cambiar a AI Token King actualizando la URL base y la clave API.

URL Base & Headers

URL Base: https://api.aitokenking.com/v1 Required headers: Content-Type: application/json Authorization: Bearer <your_api_key>

Completaciones de Chat

POST /v1/chat/completions

Crea una completación para los mensajes proporcionados. Admite todos los modelos disponibles en la plataforma.

Parámetro Tipo Requerido Descripción
model string requerido ID del modelo (p.ej. gpt-4o, claude-3-5-sonnet)
messages array requerido Array de objetos de mensaje con role and content
max_tokens integer opcional Tokens máximos a generar en la respuesta
temperature float opcional Temperatura de muestreo 0–2. Mayor = más creativo, menor = más determinista. Por defecto: 1
stream boolean opcional Transmitir respuestas parciales vía SSE. Por defecto: false
system string opcional Prompt del sistema para establecer el comportamiento y contexto del modelo

Endpoint de Modelos

GET /v1/models — Devuelve una lista de todos los modelos disponibles y su estado actual.

curl https://api.aitokenking.com/v1/models \ -H "Authorization: Bearer atk_your_key"

Cálculo de Tokens y Precios

AI Token King cobra por token consumido en entrada y salida. Los tokens se deducen de tu saldo en tiempo real. Puedes monitorear el uso desde tu panel.

Cómo se Cuentan los Tokens

  • Tokens de entrada: Cada carácter que envías — tus mensajes, prompt del sistema e historial de conversación
  • Tokens de salida: Cada carácter que el modelo genera en respuesta
  • Ambas direcciones cuentan para tu uso
  • Las conversaciones más largas acumulan tokens de contexto — los mensajes más antiguos aún cuentan como entrada

API de OpenClaw

OpenClaw es la capa de Agente de IA de AI Token King. A diferencia de las completaciones de un solo turno, los agentes OpenClaw pueden ejecutar flujos de trabajo de múltiples pasos, mantener el estado entre sesiones y activar acciones basadas en condiciones.

Resumen de OpenClaw

  • Los agentes se definen mediante una configuración de flujo de trabajo — un objeto JSON que describe pasos, condiciones y acciones
  • Cada agente tiene una sesión persistente y puede activarse mediante webhook, programación o llamada a la API
  • Los agentes usan tu saldo de tokens — cada paso consume tokens según el modelo asignado a él

Crear un Agente

POST /v1/agents

{ "name": "Customer Support Bot", "model": "gpt-4o-mini", "system_prompt": "You are a helpful customer support agent for...", "steps": [ { "id": "intake", "type": "message", "trigger": "on_start" }, { "id": "escalate", "type": "webhook", "condition": "sentiment == 'negative'", "url": "https://your-app.com/escalate" } ], "schedule": null }

Una vez creado, el agente devuelve un agent_id que puedes usar para activar ejecuciones, verificar el estado y recuperar registros a través de la API de Agentes.

SDKs y Límites de Velocidad

SDKs y Bibliotecas

Debido a que AI Token King usa una API compatible con OpenAI, puedes usar los SDKs oficiales de OpenAI apuntándolos a nuestra URL base:

# Python (OpenAI SDK) from openai import OpenAI client = OpenAI( api_key="atk_your_key", base_url="https://api.aitokenking.com/v1" ) response = client.chat.completions.create( model="gpt-4o", messages=[{"role": "user", "content": "Hello!"}] )
// Node.js (OpenAI SDK) import OpenAI from 'openai'; const client = new OpenAI({ apiKey: 'atk_your_key', baseURL: 'https://api.aitokenking.com/v1', }); const response = await client.chat.completions.create({ model: 'claude-3-5-sonnet', messages: [{ role: 'user', content: 'Hello!' }], });
